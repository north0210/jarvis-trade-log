# JARVIS Advisor 設計方針（v1.1〜v1.7 で実装済み）

> ✅ **実装状況（v1.7.0 時点）**：本設計は **v1.1〜v1.7 で実装済み**です。
> 実装範囲：9カテゴリ判定（Strong Buy / Buy / Watch / Hold / 一部利確 / Reduce / 売却候補 / Danger / 見送り）＋
> 加重合成スコア（Score/Risk/Backtest/MC/Volume/Strategy/Discipline）＋個別銘柄BTの合成反映（`/advisor`）、
> **Advisorランキング**（`/advisor-ranking`）、**お気に入り**、**Advisorスナップショット履歴/差分**、
> **AIコメント**（OFF既定・Template/外部プロバイダ・未設定時Templateフォールバック）。
> 主要ファイル：`src/lib/advisor/*`（engine/rules/context/ranking/snapshot/ai-layer 等）、`src/components/advisorView.tsx`。
>
> ⚠️ JARVIS Advisor は**自動売買ではなく判断補助**であり、**投資助言でもありません**。最終判断はユーザーが行います。
> 以下は当初の設計方針であり、用語（カテゴリ名等）は実装で一部拡張されています。

## 1. 目的

各種分析（Score / Risk / 出来高 / Strategy / Backtest / MonteCarlo / Discipline 等）を統合し、
銘柄ごとに **「買い候補」「押し目待ち」「保有継続」「一部利確候補」「損切り候補」「危険銘柄」「見送り」**
のいずれかを、根拠つきで提示する判断補助機能。売買の実行はしません。

## 2. 位置づけ・原則

- **自動売買しない**（証券口座連携もしない）
- **売買を断定しない**（「必ず上がる/下がる」等の表現は使わない）
- 出力は**候補と根拠の提示**にとどめ、最終判断はユーザー
- 既存の `alerts.ts` / `score.ts` は変更しない前提（統合レイヤーとして純関数で実装）
- 完全ローカル・外部API追加なしで成立する設計

## 3. 入力データ

| 分類 | 項目 |
|---|---|
| 銘柄指標 | Score / Grade / PER / PBR / ROE / RSI / MACD / 出来高（相対出来高・トレンド） |
| ポジション | 保有比率 / 含み損益 / 損切りライン / 利確ライン |
| 戦略・検証 | Strategy適合 / Backtest結果 / MonteCarlo結果 |
| 規律・所見 | Discipline違反 / JARVISコメント / Risk Grade |

## 4. 出力カテゴリ

1. **買い候補** — 条件が揃い、Risk許容内で新規/追加を検討できる
2. **押し目待ち** — 方向は良いが割高/過熱気味。下落待ちが無難
3. **保有継続** — 現状維持が妥当
4. **一部利確候補** — 利確ライン接近/比率過大。段階的利確を検討
5. **損切り候補** — 損切りライン到達/規律観点で縮小を検討
6. **危険銘柄** — 損切り到達・急落・重大リスクの複合
7. **見送り** — 条件不十分。手を出さない

## 5. 判定ロジック案（概念）

各カテゴリは、入力を重み付けしたルールベースのスコアリングで決定する想定（統計モデルや外部AIは使わない）。

- 買い候補：`Score(Grade A+) 且つ RSI 押し目圏 且つ Risk Grade 良好 且つ Strategy適合 且つ 出来高が枯れていない`
- 押し目待ち：`方向良好だが RSI/相対出来高が過熱しきい値以上`
- 一部利確候補：`利確ライン接近 或いは 1銘柄比率が oneStockWeightWarning 超過`
- 損切り候補：`損切りライン到達/接近 或いは 規律違反(danger) 或いは Risk悪化`
- 危険銘柄：`損切り到達 且つ (急落 或いは 重大リスク) の複合`
- 見送り：`低Score 或いは 条件不成立`

しきい値は既存 `ThresholdSettings`（relativeVolume / rsiOverheat / oneStockWeightWarning 等）を参照し、
固定値ではなくユーザー設定に基づく。

## 6. 表示例（イメージ）

```
JARVIS Advisor

買い候補：
- 7203 トヨタ
  理由：Score A / RSI 48 / PER 標準 / Risk Grade A / 出来高増加 / Strategy適合
  判断：少額候補。資金の10〜15%以内。

損切り候補：
- 〇〇〇〇
  理由：損切りライン到達 / Risk Grade C / 出来高低下 / 規律違反
  判断：売却検討。感情ではなくルールで判断してください。
```

各行に**根拠（理由）**と**行動の目安（判断）**を併記し、断定表現は避ける。
画面上部・各カード内に「本機能は投資助言ではなく判断補助です」を常時明記する。

## 7. 禁止事項（実装時の遵守）

- 自動売買しない / 証券口座連携しない
- 売買を断定しない（「必ず」「確実」等の表現禁止）
- 投資助言ではない旨を明記する
- 外部への売買データ送信・外部AIへの自動判断委譲はしない（v1.0方針の完全ローカルを維持）
